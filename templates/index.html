Efficient Algorithms

Quick Sort

Best Case: O(n log n)
Average Case: O(n log n)
Worst Case: O(n²)
Space: O(log n)

Quick Sort is a highly efficient, comparison-based algorithm that uses a divide-and-conquer strategy. It works by selecting a 'pivot' element and partitioning the array around it.

Merge Sort

Best Case: O(n log n)
Average Case: O(n log n)
Worst Case: O(n log n)
Space: O(n)

Merge Sort is a divide-and-conquer algorithm that works by dividing the input array into two halves, sorting each half, and then merging the two sorted halves.

Heap Sort

Best Case: O(n log n)
Average Case: O(n log n)
Worst Case: O(n log n)
Space: O(1)

Heap Sort is a comparison-based algorithm that uses a binary heap data structure. It works by building a max heap and repeatedly extracting the maximum element.

Simple Algorithms

Bubble Sort

Best Case: O(n)
Average Case: O(n²)
Worst Case: O(n²)
Space: O(1)

Bubble Sort repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order.

Insertion Sort

Best Case: O(n)
Average Case: O(n²)
Worst Case: O(n²)
Space: O(1)

Insertion Sort builds the final sorted array one item at a time, by repeatedly inserting a new element into the sorted portion of the array.

Selection Sort

Best Case: O(n²)
Average Case: O(n²)
Worst Case: O(n²)
Space: O(1)

Selection Sort divides the input into a sorted and unsorted region, and repeatedly selects the smallest element from the unsorted region.

Special Purpose

Radix Sort

Best Case: O(nk)
Average Case: O(nk)
Worst Case: O(nk)
Space: O(n+k)

Radix Sort is a non-comparative sorting algorithm that sorts numbers by processing individual digits from least significant to most significant.

Shell Sort

Best Case: O(n log n)
Average Case: O(n log n)
Worst Case: O(n log²n)
Space: O(1)

Shell Sort is an optimization of insertion sort that allows the exchange of items that are far apart, making the sorting more efficient.

Bitonic Sort

Best Case: O(log²n)
Average Case: O(log²n)
Worst Case: O(log²n)
Space: O(1)

Bitonic Sort is a parallel algorithm for sorting that is especially suited for implementation in hardware and parallel processor arrays.

Experimental & Fun

Bogo Sort

Best Case: O(n)
Average Case: O(n × n!)
Worst Case: O(∞)
Space: O(1)

Bogo Sort randomly shuffles the elements until they happen to be sorted. It's inefficient and meant as a joke algorithm.

Pancake Sort

Best Case: O(n)
Average Case: O(n²)
Worst Case: O(n²)
Space: O(1)

Pancake Sort sorts a disordered stack of pancakes by repeatedly flipping prefixes of the stack (like flipping pancakes on a griddle).

Gnome Sort

Best Case: O(n)
Average Case: O(n²)
Worst Case: O(n²)
Space: O(1)

Gnome Sort works by repeatedly swapping adjacent elements that are out of order, similar to how a garden gnome sorts flower pots.
